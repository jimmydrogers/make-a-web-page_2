Lesson 1: The Basics of the Web and HTML

Basics of the Web

The major components of the web

- The Internet
- Servers
- The user's Browser
- And HTTP which is the main protocol of the web

Invented in the early 1990's, the web is a collection of HTML documents, images, videos, and music. The main document is HTML.

HTML

HTML = Hyper Text Markup Language

- Text content (what you see)
- Markup (what it looks like)
- References to other documents
- Links to other pages

HTML Markup Tags

Element = <Opening Tag> CONTENTS <Closing Tag>

Examples:
Bold Tag = <b>CONTENTS</b>
M Tag (Italics/Emphasis) = <em>CONTENTS</em>

Note: Tags can have multiple attributes.

Anchor Tag <a> are for links
Example = <a href="www.reddit.com"> Click here for Reddit </a>

Image Tag <img> for images (no end tag as this is a void tag)
Example = <img src="url" alt="text">

White Space - <br> <p>

In Line Elements - <br> <b> <img> <em>

Block Elements - <p> (creates an invisible box having height and width)

Span / Div

SPAN is in line and DIV is block

<span class = "foo">CONTENTS</span>

<div class = "bar">CONTENTS</div>

HTML Document Structure

HTML Document Structure

<!DOCTYPE HTML>  (the document type is HTML)
<html>
<head>
<title>CONTENTS</title> (title of web page which is displayed at top of browser page)
 </head>
<body>
<b>CONTENTS</b>
</body>
</html>

Lesson 3: Adding CSS Style to HTML Structure

Adding Style

Add CSS style to the head section of HTML document using a link pointing to a CSS file.

<link rel= Stylesheet" href="style.css">

Example content of HTML with a CSS file:

HTML: <div class="description">Test</div>

CSS File content: .description {color : red;}

Results in the text "Test".

Alternativley you can add style without using a seperate CSS file and <link> tag by using <style> tags within your html document.

Example of CSS within HTML:

<style>

h1 { color : red}

</style>

<h1>test</h1>

Results in the text "Test".

Understanding CSS

When working on designing a web page / site there is a method called abstraction which is to find similarity or generality amongst seemly different things to avoid unnecessary repetition of work.

When tags like font, and color attributes were added to the HTML, it started a nightmare for web developers. Development of large web sites, where fonts and color information were added to every single page, became a long and expensive process. To solve this problem, CSS was created.

Inheritance is a key feature of CSS and prevents certain properties from being declared over and over again in a style sheet allowing programmers to write less code.  It is a mechanism by which properties are applied not only to a specific element but also to its descendants.  Descendant elements may inherit CSS property values from any ancestor element enclosing them.

In general descendant elements inherit test related properties. Box related properties are not inherited. Properties that can be inherited are color, font, letter spacing, line-height, list, style, text align, text indent, text transform, white space, and word spacing.

Properties that cannot be inherited are background, border display, float and clear, height and width, margin, min and max height and width, outline, overflow, padding, position, text decoration, vertical align, and Z-index.

CSS will allow you to define things once making it far more efficient than defining everything in HTML on every page which means you will type less code and your pages will be shorter and neater.

Styling Up

In CSS you first write what is called a selector which defines what element of the page a paticular style will apply to. For eaxmple you can select an HTML tag and will apply to all tags on the page such as a style for all paragraphs p {color : red;}.

Another way is to select elements by class. For example using the <div> example above in Adding Style a style can be defined by specifying the name of the class:

.description {color : red;}

Changing visual properties of an element only changes how the element is displayed. In a effort to promote consistency, a popular solution called CSS Reset found at -

The Box Revisited

The box model consists of margins, border, padding, and content.

* Content is image or text that appears on the web page.
* Padding clears an area around the content and is affected by the background color of the box. It is the space inside the box that protects the content.
* Border goes around the padding and the content and is inherited from the color properties of the box.
* Margin clears an area around the border and is transparent and can be looked at a space between boxes.

Box Sizing

The size of an element is equal to the sum of the left and right border, sum of the left and right padding, and the size of the content. This presents challenges on knowing how much box is taken up on the screen without calculating each time a change is made to a value to a border, pad, or content. An easier way to determine how much space for a box is with the use of a model CSS Style box-sizing : border-box. This  makes the calculation of the element size include both the border and padding to make layouts easier.

You can set the width of the overall box an no matter how much you tweak the border and padding sizes, the size of the box will always stay the same. For example:
max-width : 705px.

Note:
Margin size is not included.
You can specify a size using percentage of the user display instead of pixels, i.e. max-width : 50%;

Positioning Boxes

Div's are block elements so by default they take up the entire width of a page. Adding the rule display : flex to the appropiate CSS will override this behavior and let Div's appear next to each other.
Example:
.app { display : flex; }

Older Browsers

To ensure content is displayed properly on older browsers need to add browser specific prefixes to the CSS Style box and they are:
webkit-box-sizing : border-box;
moz-box-sizing : border-box;
ms-bix-sizing : border-box;
